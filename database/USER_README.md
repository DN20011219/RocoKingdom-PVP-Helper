# 用户数据文档

数据功能1：队伍表

队伍表存储了用户的一支队伍的全部配置。包括精灵列表和携带的共鸣魔法。队伍有他自己的队伍名。

精灵列表包含多个精灵，对每个精灵有它的各项属性值、携带的技能、血脉。

共鸣魔法是枚举类型，目前只有“进化之力”和“愿力强化”两种，后续可能拓展更多种。

但是这个导入格式还缺少了每个精灵的六种属性，因此需要提供额外的设置接口来设置每个精灵的六种属性，为了简单起见，设置接口输入可以为阵容名 + 精灵名 + 六个属性值。

我已经把这套功能补成了数据库接口，使用方式如下：

1. 先把整段阵容文本通过 `POST /teams/import` 导入，文本里会自动解析队伍名、共鸣魔法、精灵名、血脉和四个技能。
2. 再通过 `POST /teams/{team_name}/pets/{pet_name}/attributes` 给某个精灵补录六维属性，属性顺序固定为：生命、物攻、魔攻、物防、魔防、速度。

例如，补录“圣羽翼王”属性时，请发送：

```json
{
	"生命": 233,
	"物攻": 201,
	"物防": 174,
	"魔攻": 167,
	"魔防": 180,
	"速度": 267
}
```

如果要查看导入后的结果，可以直接调用 `GET /teams/{team_name}`。
